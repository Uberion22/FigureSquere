В качестве реализации выбран следующий путь при котором две геометрические фигуры реализуют один интерфейс для вычисления площади нахождения периметра (длинны окружности).Также можно было просто реализовать методы с разной сигнатурой для одного класса с расчетами нужных характеристик.  Для классов обеих фигур добавлены простые валидации с генерацией исключений (свой класс исключений не добавлен) . Добавлено несколько тестов на проверку корректности расчетов и генерацию исключений.
Что касается вопроса о расчете площади фигуры без знания ее типа: можно использовать  подход, при котором фигура строится путем определения координат ее вершин на плоскости. В этом случае можно рассчитать площадь. Но этот метод не универсален. 

Запрос, возможно,  выглядит так:
select p.ProductName, c.CategoryName from Products as p
left join Category as c on p.Id = c.ProductId
group by p.ProductName
